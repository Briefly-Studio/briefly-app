# Briefly v2.0 Kickoff

## Purpose
v2 focuses on learning quality and long-term retention improvements that require real user signals. v1.5 is frozen and stable.

## Non-Goals (not in v2.0)
- No accounts / auth
- No cloud sync
- No charts unless data volume justifies it
- No AI automation before manual signals exist

## v2.x Roadmap (Sequenced)

### v2.1 — Difficulty Tagging (Manual Signals)
Goal: Allow users to label cards with difficulty (Easy / Medium / Hard) to build trustworthy learning signals.
Deliverables:
- Card model supports difficulty tag (optional)
- Edit card UI includes difficulty selector
- Review/Quiz can display tag optionally
- Storage migration safe (backward compatible)

### v2.2 — Smart Shuffle (Weighted Review)
Goal: Prioritize weak, difficult, or stale cards during review.
Inputs:
- Difficulty tag
- Recency (last studied)
- Miss history (future if review gets grading)
Deliverables:
- Weighted ordering algorithm
- Toggle: Smart Shuffle on/off per deck (optional)

### v2.3 — Deck Export (Trust & Portability)
Goal: Give users ownership of their data.
Deliverables:
- Export deck + cards as JSON
- Optional CSV export for cards
- Optional PDF export (read-only share)

## Success Criteria
- Users can self-identify difficult material quickly.
- Review sessions feel more effective without adding friction.
- App remains fast, calm, and predictable.

## Risks
- Over-complicating UI with “power features”
- Premature automation reduces trust
- Breaking storage compatibility

## Next Step
Implement v2.1 Difficulty Tagging end-to-end (model → storage → UI).
